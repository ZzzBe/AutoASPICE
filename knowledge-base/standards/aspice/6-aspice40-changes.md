# Automotive SPICE 4.0 - 变更对比

## 概述

本文档详细记录了 ASPICE 4.0 与 ASPICE 3.1 之间的主要差异，包括新增流程、术语变更、Work Products 更新等内容。

---

## 版本信息

| 项目 | ASPICE 3.1 | ASPICE 4.0 |
|------|-----------|------------|
| 发布年份 | 2017 | 2023 |
| 流程总数 | 约31个 | 33个 |
| 新增流程组 | - | MLE, HWE |
| 评估范围 | VDA Scope | 新的流程组合方式 |

## 流程领域变更

### 新增流程组

#### MLE - 机器学习工程 (Machine Learning Engineering)

ASPICE 4.0 首次引入机器学习工程流程组，涵盖汽车软件中 ML 模块的开发：

| 流程 ID | 流程名称 | 目的 |
|---------|---------|------|
| MLE.1 | 机器学习需求分析 | 从软件需求细化机器学习需求，定义数据特征和 ODD |
| MLE.2 | 机器学习架构 | 定义 ML 模型、预处理/后处理、超参数范围 |
| MLE.3 | 机器学习训练 | 使用训练数据训练 ML 模型，调优超参数 |
| MLE.4 | 机器学习模型测试 | 验证 ML 模型符合 ML 需求 |

## 术语变更

### 核心术语更新

| ASPICE 3.1 | ASPICE 4.0 | 说明 |
|------------|------------|------|
| Testing (测试) | **Verification (验证)** | 所有 SWE.4, SWE.5, SWE.6 等测试相关术语更新为验证 |
| Stakeholder | **受影响方** | 能力等级 1 级使用 |
| - | **参与方** | 能力等级 2 级使用 |

### 新增关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **机器学习 (ML)** | Machine Learning | 描述软件从特定训练数据中学习并将这些知识应用于其他类似任务的能力 |
| **超参数** | Hyperparameter | 在机器学习中控制 ML 模型训练的参数值（如学习率、损失函数、模型深度等） |
| **运行设计域 (ODD)** | Operational Design Domain | 给定的整体系统或其功能被专门设计用于工作的运行条件 |
| **硬件组件** | Hardware Component | 为实现某一功能的硬件元器件逻辑性或物理性组合 |
| **硬件元器件** | Hardware Element | 基本硬件要素，其用途和功能不可进一步细分或切割 |
| **信息需要** | Information Need | 用以表征与过程或产品相关的有效性和效率的需要 |

### Measure 与 Metric 的区分

ASPICE 4.0 明确区分了两个关键概念：

```
┌─────────────────────────────────────────────────────┐
│  Measure (措施)                                      │
│  ────────────────────                               │
│  定义: 为达成某种意图的活动                           │
│  例子: 培训、代码评审、测试                          │
│                                                     │
│  特点:                                              │
│  - 具体的行动或活动                                  │
│  - 可执行、可操作                                    │
│  - 直接产生结果                                      │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  Metric (度量项)                                     │
│  ────────────────────                               │
│  定义: 与既定的信息需要相匹配的定量或定性的            │
│        可衡量指标                                    │
│  例子: 缺陷密度、代码覆盖率、进度偏差率               │
│                                                     │
│  特点:                                              │
│  - 可量化的指标                                      │
│  - 支持决策                                         │
│  - 用于监控和评估                                    │
└─────────────────────────────────────────────────────┘
```

## 评估范围变更

### VDA Scope 的移除

ASPICE 4.0 中不再使用 "VDA Scope" 概念，采用新的流程组合方式：

**ASPICE 3.1 VDA Scope (典型范围)**:
- SYS.2, SYS.3, SYS.4, SYS.5
- SWE.1, SWE.2, SWE.3, SWE.4, SWE.5, SWE.6
- SUP.1, SUP.8, SUP.9, SUP.10
- MAN.3

**ASPICE 4.0 新的评估范围方式**:

评估范围现在根据项目特性和产品类型动态确定：

```
┌─────────────────────────────────────────────────────┐
│            评估范围确定流程                           │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 产品类型识别                                     │
│     ├─ 纯软件产品 → SYS + SWE 流程组                 │
│     ├─ 软硬件产品 → + HWE 流程组                     │
│     ├─ 含 ML 功能 → + MLE 流程组                     │
│     └─ 独立系统 → + VAL 流程                         │
│                                                     │
│  2. 开发角色识别                                     │
│     ├─ 供应商角色 → SPL 流程组                       │
│     ├─ 采购方角色 → ACQ 流程组                       │
│     └─ 两者兼顾 → SPL + ACQ                         │
│                                                     │
│  3. 组织成熟度考量                                   │
│     ├─ 初始评估 → 核心流程 (SYS, SWE, SUP, MAN)       │
│     ├─ 进阶评估 → + PIM, REU                        │
│     └─ 高级评估 → + MAN.5, MAN.6                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 新流程详细说明

### MLE.1: 机器学习需求分析

**目的**: 从软件需求中细化出机器学习相关的需求，包括数据需求和运行设计域定义。

**Base Practices**:

| BP | 描述 |
|----|------|
| BP1 | 识别需要 ML 实现的软件需求 |
| BP2 | 定义 ML 功能需求（输入、输出、预期行为） |
| BP3 | 定义数据需求（数据类型、数量、质量要求） |
| BP4 | 定义 ODD（运行条件、边界条件） |
| BP5 | 建立与软件需求的双向追溯 |
| BP6 | 沟通并评审 ML 需求 |

**Work Products**:
- 19-01: ML 需求规格说明
- 19-02: 数据需求说明
- 19-03: ODD 定义文档
- 13-22: 追溯记录

### MLE.2: 机器学习架构

**目的**: 定义 ML 模型架构、预处理/后处理流程和超参数范围。

**Base Practices**:

| BP | 描述 |
|----|------|
| BP1 | 选择合适的 ML 模型类型 |
| BP2 | 定义模型架构（层数、节点数、激活函数等） |
| BP3 | 定义预处理流程（数据清洗、特征提取、归一化等） |
| BP4 | 定义后处理流程（结果解释、阈值判断等） |
| BP5 | 定义超参数范围 |
| BP6 | 建立与 ML 需求的追溯 |

**Work Products**:
- 04-51: ML 架构文档
- 01-54: 超参数定义
- 19-04: 预处理/后处理规格说明

### MLE.3: 机器学习训练

**目的**: 使用训练数据训练 ML 模型并调优超参数。

**Base Practices**:

| BP | 描述 |
|----|------|
| BP1 | 准备训练数据（采集、标注、增强） |
| BP2 | 划分数据集（训练集、验证集、测试集） |
| BP3 | 执行模型训练 |
| BP4 | 调优超参数 |
